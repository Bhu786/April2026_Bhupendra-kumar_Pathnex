# 🚀 Git & GitHub Notes (Interview Ready)

---

## 🔹 Git vs GitHub

### Git
- Open source tool
- Distributed Version Control System (DVCS)
- Code changes track karta hai
- Multiple developers collaboration possible
- Internet ke bina bhi kaam karta hai

### GitHub
- Remote repository platform
- Git ka use karta hai
- Collaboration (PR, Issues, etc.) ke liye use hota hai

---

## 🔹 Basic Git Commands

```bash
git init              # Initialize repository
git add .             # Add all files to staging area
git commit -m "msg"   # Save changes to local repo
````

👉 Ye sab commands **local system** par run hote hain

---

## 🔹 Push Code to Remote

```bash
git push origin main
```

* Code ko GitHub par bhejne ke liye use hota hai

---

## 🔹 Branching

* Branch ek **copy hoti hai parent code ki**

```bash
git branch feature     # Create branch
git checkout feature   # Switch branch
git checkout -b feature  # Create + switch
```

---

## 🔹 Git Areas Flow

```
Working Directory → Staging Area → Local Repository
```

* `git add` → Staging Area
* `git commit` → Local Repository

---

## 🔹 Useful Commands

```bash
git status   # Tracked & untracked files check
git add .    # Dot means all files
```

---

## 🔹 Unstage File

```bash
git restore --staged <file-name>
```

👉 Staging se file wapas working directory me aa jati hai

---

## 🔹 Reset Commands

| From              | To | Command                     |
| ----------------- | -- | --------------------------- |
| Commit → Staging  |    | `git reset --soft HEAD~1`   |
| Commit → Working  |    | `git reset HEAD~1`          |
| Staging → Working |    | `git restore --staged file` |

### Types of Reset

* **Soft** → Sirf commit undo, code safe
* **Mixed (default)** → Staging hata deta hai
* **Hard** → Sab kuch delete (danger ⚠️)

---

## 🔹 Revert vs Reset

| Feature  | Reset            | Revert                       |
| -------- | ---------------- | ---------------------------- |
| Use Case | Local changes    | Remote/pushed code           |
| History  | Change karta hai | Safe (new commit banata hai) |

# 🎯 Interview Questions & Answers

---

## ❓ 1. Rebase vs Merge

### Merge

* 2 branches ko combine karta hai
* New commit create hota hai (merge commit)
* History safe rehti hai

### Rebase

* Linear history banata hai
* Commit ko rewrite karta hai
* Clean history but risky if already pushed

👉 **Interview Tip:**

* Team projects → Merge
* Clean history → Rebase

---

## ❓ 2. Hotfix kya hota hai?

* Production bug ko jaldi fix karna
* Direct main/master branch se new branch banake fix kiya jata hai

```bash
git checkout -b hotfix
```

---

## ❓ 3. Cherry Pick kya hai?

* Kisi specific commit ko ek branch se dusre branch me lana

```bash
git cherry-pick <commit-id>
```

👉 Useful jab ek hi change chahiye, pura branch nahi

---

## ❓ 4. Branch Strategy kaunsi use karte ho?

### Common Strategies:

* **Git Flow**

  * main → production
  * develop → working
  * feature → new features
  * hotfix → urgent fix

* **Trunk Based**

  * Single main branch
  * Fast development

👉 Interview me bolo:

> "I prefer Git Flow for structured projects and Trunk-based for fast agile teams."

---

## ❓ 5. .gitignore kya hai?

* Unwanted files ko ignore karta hai

Example:

```
node_modules/
.env
*.log
```

👉 Ye files repo me push nahi hoti

---

## ❓ 6. Clone vs Fork

| Clone                 | Fork                     |
| --------------------- | ------------------------ |
| Local copy banata hai | Remote copy (GitHub pe)  |
| `git clone URL`       | GitHub UI se fork        |
| Direct kaam ke liye   | Open-source contribution |

---

## ❓ 7. Revert vs Reset (Interview Style)

* **Reset** → History change karta hai (local use)
* **Revert** → New commit banata hai (safe for remote)

👉 Best Practice:

> "Never use reset on pushed code, use revert."

---

# ✅ Quick Revision (5 Line Summary)

* Git → Version control, GitHub → Remote platform
* `git add` → staging, `git commit` → save
* Branch → code copy for safe development
* Reset → undo changes, Revert → safe undo
* Merge → safe, Rebase → clean history
