# ☁️ 1. Storage Services

## 🔹 a) Blob Storage (== AWS S3)

**Definition:**
Object storage service used to store unstructured data.

**Use Cases / Practical:**

* Store logs, images, videos
* Backup and archival data
* Static website hosting

---

## 🔹 b) File Storage (== AWS EFS)

**Definition:**
Managed file storage service that can be accessed by multiple servers.

**Use Cases / Practical:**

* Multiple servers sharing same files
* Configuration sharing

**Example:**

* 5 servers running the same app
* All need same `config.json`
* Store in File Storage → all servers access it

---

## 🔹 c) Managed Disks (== AWS EBS)

**Definition:**
Persistent block storage attached to virtual machines.

**Use Cases / Practical:**

* Store OS of VM (Ubuntu, Windows)
* Store database data (MySQL, MongoDB)
* Store application files

**Daily Tasks:**

* Resize disks
* Take snapshots (backup)
* Attach / Detach disks

---

# 🌐 2. Networking

## 🔹 a) VNet (== AWS VPC)

**Definition:**
Private network environment in cloud.

**Use Cases / Practical:**

* Separate **Dev & Prod environments**
* Isolate applications securely

---

## 🔹 b) Subnet (== AWS Subnet)

**Definition:**
Smaller network inside VNet.

**Use Cases / Practical:**

* Divide architecture into layers:

  * Frontend
  * Backend
  * Database

---

## 🔹 c) Network Security Group (== AWS Security Group)

**Definition:**
Firewall that controls inbound & outbound traffic.

**Use Cases / Practical:**

* Allow only specific ports (e.g., 80, 443)
* Restrict access to specific IPs
* Secure applications

---

# 🔐 3. Security Services

## 🔹 Azure AD (== AWS IAM)

**Definition:**
Identity and Access Management service.

**Use Cases / Practical:**

* Manage users, roles, permissions
* Secure access to resources

---

## 🔹 Key Vault (== AWS Secrets Manager / KMS)

**Definition:**
Secure storage for sensitive data.

**Use Cases / Practical:**

* Store API keys, passwords, certificates
* Avoid hardcoding secrets in code

# 🔁 4. Azure DevOps

**Definition:**
Complete platform for **planning, coding, building, testing, deploying, and monitoring** applications.

## 🔄 Lifecycle:

Plan → Code → Build → Artifact → Test → Deploy → Monitor

## 🧰 Tools Mapping:

| Step     | Azure Tool     | AWS Equivalent       |
| -------- | -------------- | -------------------- |
| Plan     | Azure Boards   | Jira (No AWS direct) |
| Code     | Azure Repos    | CodeCommit / GitHub  |
| Build    | Pipelines (CI) | CodeBuild            |
| Artifact | Artifacts      | S3 / Artifact Repo   |
| Test     | Test Plans     | 3rd Party Tools      |
| Deploy   | Pipelines (CD) | CodeDeploy           |
| Monitor  | Logs/Feedback  | CloudWatch / 3rd     |

---

## 🔹 CI/CD Pipelines

**Definition:**
Automated process to build, test, and deploy code.

**Use Cases / Practical:**

* Automatic deployment after code push
* Faster and reliable releases

---

# 📊 5. Monitoring

## 🔹 Azure Monitor (== AWS CloudWatch)

**Definition:**
Service to collect logs and metrics.

**Use Cases / Practical:**

* Monitor CPU, memory, errors
* Create alerts (e.g., server down)
* Build dashboards

# ⚙️ 6. Containers & Kubernetes

## 🔹 AKS (Azure Kubernetes Service) == AWS EKS

**Use Cases / Practical:**
