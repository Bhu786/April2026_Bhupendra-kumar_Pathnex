# 🐳 Docker Complete Notes

## 🔹 What is Docker?

* Docker is an **open-source platform** used to build, ship, and run applications in **containers**.
* It ensures applications run **the same way everywhere** (dev, test, production).
* Uses the **host OS kernel**, making it **lightweight and portable**.

---

## 🔹 Container

* A **container** is a **running instance of a Docker image**.
* It contains everything needed to run an application.

---

## 🔹 Dockerfile

* A **Dockerfile** is a script with a set of instructions to build a Docker image.

### Common Instructions:

```dockerfile
FROM       # Base image
RUN        # Execute commands
CMD        # Default command
COPY       # Copy files
ENV        # Environment variables
EXPOSE     # Port exposure
WORKDIR    # Set working directory
```

---

## 🔹 Docker Image

* A **read-only template** used to create containers.
* Contains:

  * Code
  * Runtime
  * Libraries
  * Environment variables
  * Config files

👉 One image can create **multiple containers**.

---

## 🔹 Docker Engine

* Core component that runs Docker.

### Components:

* **Docker CLI** → Command line interface
* **Docker API** → Communication layer
* **containerd** → Handles container lifecycle (modern replacement of `dockerd` internals)

---

## 🔹 Docker Compose

* Tool to **run multiple containers together** using a single `docker-compose.yml` file.

👉 Useful for:

* Microservices
* Multi-container apps (e.g., frontend + backend + DB)

---

## 🔹 Docker Registry

* Stores Docker images (like Play Store for apps).

### Types:

* **Public**: Docker Hub
* **Private**:

  * Nexus
  * AWS ECR (Elastic Container Registry)

---

## 🔹 Volume

* Used for **persistent storage**.

👉 Problem:

* Container deleted → data lost ❌

👉 Solution:

* Use **volumes** → data stored outside container ✅

📌 Example:

* Local system storage
* AWS EC2 → stored in EBS

---

## 🔹 Networking in Docker

### Default Network Types:

* **bridge** → Default network for containers
* **overlay** → Used in Swarm for multi-host communication
* **none** → No network

---

## 🔹 Docker Swarm vs Kubernetes

| Feature    | Docker Swarm | Kubernetes |
| ---------- | ------------ | ---------- |
| Setup      | Easy         | Complex    |
| Scaling    | Basic        | Advanced   |
| Use Case   | Small apps   | Enterprise |
| Popularity | Low          | High       |

---

## 🔹 Docker Workflow

```
Dockerfile → Docker Image → Docker Container
```

---

## 🔹 Important Commands

```bash
docker build     # Build image
docker run       # Run container
docker ps        # List running containers
docker ps -a     # List all containers
docker stop      # Stop container
docker rm        # Remove container
docker rmi       # Remove image
docker pull      # Download image
docker push      # Upload image
docker logs      # View logs
docker restart   # Restart container
```

---

## 🔹 System Command

```bash
systemctl status docker
```

---

## 🔹 Interview Points 🚀

* Docker uses **host kernel → lightweight**
* **Image = blueprint**, **Container = running app**
* One image → **multiple containers**
* **Volumes = data persistence**
* **Compose = multi-container management**
* **Registry = image storage**
* **Swarm/K8s = container orchestration**
