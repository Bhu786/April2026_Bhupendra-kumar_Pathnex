# 🚀 Helm (Kubernetes Package Manager)

## 🔹 What is Helm?
Helm is a package manager for Kubernetes (just like Play Store for apps).

👉 Package Manager = Tool to install, update, and manage applications easily.

---

## 🔹 Why Helm?
Without Helm:
- Managing multiple YAML files is painful
- Manual deployment is complex

With Helm:
- Automates and simplifies Kubernetes deployments
- One command can deploy entire application

Example:
10 YAML files → 1 Helm Chart  
1 command → helm install

---

## 🔹 Helm Chart
Helm Chart = Collection of YAML templates + configuration + metadata

Structure:
my-chart/
│── Chart.yaml  
│── values.yaml  
│── templates/  
│── charts/  

### 📁 templates/
👉 Contains actual Kubernetes YAML files (Deployment, Service, etc.)

### 📁 values.yaml
- Default configuration file
- Not mandatory but recommended
- Helps in clean & environment-based config

# 🚀 Kubernetes Basics

## 🔹 How to Expose a Pod?
1. Create Pod  
2. Add Label  
3. Expose using NodePort  
4. Access via NodeIP:Port  

---

## 🔹 How to Check Logs?
- kubectl logs <pod-name>
- kubectl logs -f <pod-name>  (live logs)
- kubectl logs <pod-name> -c <container-name> (multi-container)

---

## 🔹 How Service Works?
- Service selects Pods using labels
- Provides stable IP
- Load balances traffic

Types:
- ClusterIP
- NodePort
- LoadBalancer

# 🚀 Important kubectl Commands

## 🔹 Basic Commands
- kubectl get ns
- kubectl get pods
- kubectl get svc
- kubectl get nodes

- kubectl logs <pod-name>
- kubectl describe pod <pod-name>

---

## 🔹 Practical Commands

1. Create Namespace
kubectl create ns dev

2. List Namespaces
kubectl get ns

3. Create Pod (default namespace)
kubectl run nginx-pod --image=nginx

👉 Use case:
- Testing
- Debugging
- Quick pod creation without YAML

4. Check Pods
kubectl get pods

5. Create Pod in Specific Namespace
kubectl run nginx-pod2 --image=nginx -n dev

6. Create Pod with Label
kubectl run nginx-pod3 --image=nginx --labels="app=my-nginx-app"

👉 Labels are required for Service selectors

7. Check Services
kubectl get svc

8. Expose Pod as NodePort
kubectl expose pod nginx-pod3 --type=NodePort --port=80 --name=my-first-service

👉 Use case:
- Access pod from browser
- External access via NodeIP:NodePort

9. Verify Services
kubectl get svc

10. Get Detailed Pod Info
kubectl get pods -o wide

👉 Use case:
- View Pod IP
- Node info
- Debug networking
