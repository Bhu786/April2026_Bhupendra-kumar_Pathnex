## ☁️ Storage Services

### 🔹 S3 (Simple Storage Service)

* Remote object storage (like Google Drive)
* Used for:

  * Files, images, videos
  * Backups (snapshots)
  * Static websites
* Highly scalable & durable

---

### 🔹 EBS (Elastic Block Store)

* Acts like **hard disk for EC2**
* Data persists even after EC2 shutdown

#### 📌 How it works:

1. OS loads from EBS
2. Application runs from EBS
3. Data read/write happens in EBS
4. Logs stored in EBS

#### 📌 Use cases:

* Databases (MySQL, MongoDB)
* Transaction data
* Fast disk requirement systems

#### 📌 Extra:

* Snapshots (backup) → stored in S3

---

### 🔹 EFS (Elastic File System)

* Shared file system for multiple EC2 instances
* Used when **multiple servers need same data access**

---

### 🧠 Memory Trick

* **EBS** → Server chalane ke liye
* **S3** → Data store karne ke liye
* **EFS** → Sharing ke liye

## 🌐 Networking

### 🔹 VPC (Virtual Private Cloud)

* Isolated network in AWS region

---

### 🔹 Subnet

* Smaller network inside VPC

#### Types:

* **Public Subnet**

  * Has route to Internet Gateway (IGW)
* **Private Subnet**

  * No direct route to IGW

👉 Interview line:

> Public subnet mein IGW route hota hai, private mein nahi.

---

### 🔹 Internet Gateway (IGW)

* Connects VPC to internet
* **2-way traffic (inbound + outbound)**
* Free
* Used in public subnet

---

### 🔹 NAT Gateway

* Allows **only outbound traffic**
* Used for private subnet
* Paid service
* Must be placed in public subnet
* Connected to IGW

---

### 🆚 IGW vs NAT

| Feature | IGW                | NAT Gateway    |
| ------- | ------------------ | -------------- |
| Traffic | Inbound + Outbound | Only Outbound  |
| Cost    | Free               | Paid           |
| Use     | Public subnet      | Private subnet |

---

## 🔗 VPC Peering

* Direct private connection between two VPCs

### 📌 Types:

#### 1. Mesh Peering

* Every VPC connected to every other VPC
* Complex & hard to manage

#### 2. Hub & Spoke

* One central VPC (Hub)
* Multiple VPCs (Spokes)
* Easy control & scalable

---

### ❗ When NOT to use VPC Peering:

* Same CIDR range (not allowed)

---

## 🌍 Route 53
