## 📋 Linux Process Management — Presentation Notes

---

### 1️⃣ `top` Command — See All Running Processes

The `top` command shows **all processes currently running** on your system.

- You can see **which process is using too much CPU or memory**
- If something is overloading the system — you can **kill it** to free up resources

> *"Think of it like Task Manager in Windows — but for Linux."*

---

### 2️⃣ Load Average — Important for Interviews!

When you run `top`, you see this:

```
Load Average: 6.28, 5.05, 4.24
```

| Value | Meaning |
|---|---|
| 6.28 | Last **1 minute** |
| 5.05 | Last **5 minutes** |
| 4.24 | Last **15 minutes** |

> This tells you **how busy your system was** over time.

---

### 3️⃣ CPU Core Usage — Can Go Above 100%!

- If your system has **4 cores**, total CPU can show **400%**
- Example: 2 cores fully used = **200%**
- Going too high = **harmful for the system** ⚠️

---

### 4️⃣ Two Most Important Commands for Interviews

```bash
top        # Live CPU & memory usage
free -h    # Shows RAM usage in readable format
```

> *"If interviewer asks how to check CPU — these two are your answer."*

---

### 5️⃣ `ps` Command — Snapshot of Processes

```bash
ps aux                        # Show all running processes
ps aux | grep -i studio       # Search for a specific process
```

- `top` is **live/real-time**
- `ps` is a **snapshot** — like a photo at that moment

---

### 6️⃣ `kill` Command — Stop a Process

```bash
kill <PID>        # Normal kill
kill -9 <PID>     # Force kill (use when process is stuck)
```

> `-9` means **forcefully terminate** — no questions asked

```bash
pkill "Visual Studio"    # Kill by name
pkill "Code"
```

---

### 7️⃣ `pgrep` — Find Process by Name

```bash
pgrep <process_name>    # Returns PID of that process
```

> Use this **before kill** — to find the PID first

---

### 8️⃣ `ls` + `grep` — Search in Files/Folders

```bash
ls -la | grep <name>      # Search (case sensitive)
ls -la | grep -i <name>   # Search (ignore case) ✅
```

> `-i` flag = **case insensitive** — safer to use always
