# 📘 AWS Notes (Quick Revision)

---

## ☁️ AWS (Cloud Service Provider)

AWS ek **cloud service provider** hai jo 3 types ki services deta hai:

### 🔹 1. IaaS (Infrastructure as a Service)

* Sab kuch khud manage karna hota hai:

  * OS
  * Runtime
  * Application
  * Data
* Cloud provider sirf **infrastructure (VM, storage, network)** deta hai

**Examples:**

* AWS EC2
* Azure VMs

👉 *Short line:*
**"IaaS mein sab kuch user handle karta hai."**

---

### 🔹 2. PaaS (Platform as a Service)

* Cloud provider kuch part ready deta hai:

  * OS
  * Runtime
* User ko sirf:

  * Application
  * Data handle karna hota hai

**Examples:**

* Heroku

👉 *Short line:*
**"PaaS mein sirf app aur data manage karna hota hai."**

---

### 🔹 3. SaaS (Software as a Service)

* Sab kuch cloud provider handle karta hai
* User sirf use karta hai

**Examples:**

* Gmail
* Netflix
* Zoom
* YouTube

👉 *Short line:*
**"SaaS mein bas use karo, kuch manage nahi karna."**

---

## 🔐 IAM (Identity and Access Management)

AWS services ko secure tarike se manage karta hai.

### 🔹 Root User

* Account banate time jo email/password use hota hai
* Full access (dangerous)

👉 *Interview line:*
**"Root user has unrestricted access."**

---

### 🔹 IAM User

* Individual login for developers/admins
* Username + password / access keys

---

### 🔹 IAM Group

* Users ka collection
* Ek policy attach → sabko same permission

👉 Example: Developers, Admins

---

### 🔹 Role

* Temporary access ke liye use hota hai
* Password nahi hota
* Temporary credentials milte hain

👉 Used by:

* EC2
* Lambda
* Cross-account access

---

### 🔹 Policies

* JSON document
* Define karta hai:

  * Effect (Allow/Deny)
  * Action (kya kar sakte ho)
  * Resource (kis pe kar sakte ho)

---

## 📈 Scaling

### 🔹 Horizontal Scaling

* Instances ki **number increase**
* Example: 1 server → 10 servers

---

### 🔹 Vertical Scaling

* Same instance me resources increase
* Example: RAM, CPU badhana

---

---

### 🔹 Region-Specific Services

## 🌐 VPC (Virtual Private Cloud)

Private network inside AWS

---

### 🔹 Subnet

* VPC ka chhota part
* Types:

  * Public Subnet
  * Private Subnet

---

### 🔹 Internet Gateway (IGW)

* Internet ka **main gate**
* Inbound + Outbound (2-way)
* Free
* Public subnet ke liye

---

### 🔹 NAT Gateway

* Sirf **outbound access** (1-way)
* Private subnet ke liye
* Paid
* Public subnet me deploy hota hai
* IGW se connected hota hai

---

## 🎯 Interview One-Liners

### 🔸 Public vs Private Subnet

👉 **"Public subnet mein IGW route hota hai, private mein nahi."**

---

### 🔸 IGW vs NAT Gateway

| Feature | IGW           | NAT Gateway    |
| ------- | ------------- | -------------- |
| Access  | 2-way         | 1-way          |
| Cost    | Free          | Paid           |
| Use     | Public subnet | Private subnet |

---

## ⚠️ Important Correction (VERY IMPORTANT)

❌ Tumne likha tha:

* Horizontal = resource increase
* Vertical = instance increase

✅ **Correct hai:**

* **Horizontal scaling → instances badhana**
* **Vertical scaling → resources badhana**
